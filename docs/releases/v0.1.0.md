# dropboard v0.1.0

The first public preview of dropboard is ready. It gives AI agents a simple way
to publish substantial work outside the chat window, and gives you a private,
mobile-friendly place to review and organize it.

## Highlights

- **A real inbox for AI deliverables**: Unread states, search, category filters,
  pinning, temporary previews, archive, trash, and undo make generated work feel
  manageable instead of disposable.
- **Living documents without duplicates**: Stable document keys update one
  document through immutable revisions. You can inspect its history, prevent
  stale writes, and restore an earlier version without losing what came after.
- **An archive you can find things in**: Edit tags, file documents by project
  and nested folder, reuse existing organization with suggestions, and move up
  to 500 selected documents together.
- **Works with your agents**: The included CLI, REST API, Claude Code skill,
  Codex guide, and generic prompt cover common publishing workflows without
  tying the board to one AI vendor.
- **Private by default**: PIN sessions protect the UI, bearer tokens protect
  write APIs, artifacts render inside a restricted sandbox, and revocable share
  links expose only the document you choose.

## Self-hosting

Run dropboard from source with Node.js 20.9 or newer, or use the included Docker
Compose setup with a persistent data volume. No database or external storage
service is required.

This release is distributed as source and Docker configuration; the root
package is not published to npm.

## Quality

- Verified on Node.js 20 and 24.
- 50 automated logic and API checks.
- Browser tests cover bulk organization, revision restoration, and permanent
  deletion.
- Production dependency audit reports no known vulnerabilities at release time.

## Upgrade notes

This is the first tagged release, so there are no migrations from an earlier
release. Existing source installations can pull `v0.1.0`, run `npm ci`, build
with `npm run build`, and restart the service. Existing filesystem items remain
compatible.
